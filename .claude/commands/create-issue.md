# GitHub Issue 作成

新しいIssueを対話的に作成します。

## 手順

1.  **Issueの内容をヒアリング**
    *   ユーザーに以下の情報を確認してください：
        *   **タイトル**（必須）: Issueの件名
        *   **説明**（必須）: 何を実装/修正するか
        *   **種類**: `feat`（機能追加）, `fix`（バグ修正）, `refactor`（リファクタリング）, `docs`（ドキュメント）, `test`（テスト）
        *   **優先度**: `High`, `Medium`, `Low`
        *   **関連Issue**（任意）: 依存関係のあるIssue番号

2.  **Issue本文のテンプレート作成**
    *   以下の形式でIssue本文を作成してください：

    ```markdown
    ## 概要
    {ユーザーが入力した説明}

    ## 背景・目的
    {なぜこの変更が必要か}

    ## 実装内容
    - [ ] {具体的なタスク1}
    - [ ] {具体的なタスク2}
    - [ ] {具体的なタスク3}

    ## 受け入れ条件
    - [ ] {完了条件1}
    - [ ] {完了条件2}

    ## 関連Issue
    {関連するIssue番号があれば記載}

    ## 優先度
    {High/Medium/Low}
    ```

3.  **ユーザーへの確認**
    *   作成するIssueの内容を表示し、「この内容でIssueを作成しますか？」と確認してください。
    *   修正が必要な場合は、ユーザーの指示に従って修正してください。

4.  **Issueの作成**
    *   承認が得られたら、以下のコマンドでIssueを作成してください。
    *   `gh issue create --title "{タイトル}" --body "{本文}"`
    *   ラベルがあれば `--label "{ラベル}"` を追加

5.  **ロードマップへの追加確認**（Issue 125 が存在する場合）
    *   「Issue 125（ロードマップ）に追加しますか？」と確認してください。
    *   承認された場合：
        *   `gh issue comment 125 --body "新規Issue #{作成したIssue番号} を追加: {タイトル}"`

6.  **完了報告**
    *   作成されたIssueのURLを表示してください。
    *   「Issue #{番号} を作成しました: {URL}」

## ショートカット

引数が渡された場合は、それをタイトルとして使用し、説明のみをヒアリングしてください。

例: `/create-issue ユーザー認証機能の追加`
→ タイトル「ユーザー認証機能の追加」として、説明からヒアリング開始

## ラベル対応表

| 種類 | ラベル |
|------|--------|
| feat | enhancement |
| fix | bug |
| refactor | refactoring |
| docs | documentation |
| test | test |
