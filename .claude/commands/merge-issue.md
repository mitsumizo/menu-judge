# Issue完了とマージ (Issue 125 更新)

現在の機能ブランチでの作業を完了し、PRの作成・マージ・Issueクローズ・ロードマップ更新を一括で行います。

## 手順

1.  **Issue IDの特定**
    *   現在のブランチ名を確認し、Issue番号を特定してください。
    *   ブランチ名の命名規則: `feature/issue-{Issue番号}-{タイトル}`

2.  **変更のプッシュ**
    *   現在のブランチの未プッシュのコミットをリモートにプッシュしてください。
    *   `git push origin HEAD`

3.  **PRの確認と作成**
    *   現在のブランチに関連するPull Requestが存在するか確認してください。
    *   存在しない場合は、`gh pr create --fill` で作成してください（タイトルや内容は必要に応じてユーザーに確認）。

4.  **マージの実行**
    *   ユーザーにマージして良いか確認した後、PRをマージしてください。
    *   `gh pr merge --merge --delete-branch`
    *   ※ マージがブロックされている場合（CI落ちやレビュー待ちなど）は、ユーザーに状況を伝えて中断してください。

5.  **Issueのクローズ**
    *   実装していたIssue (手順1で特定した番号) をクローズしてください。
    *   `gh issue close {Issue番号}`

6.  **Issue 125（ロードマップ）の更新**
    *   Issue 125に対して完了報告のコメントを追加してください。
    *   `gh issue comment 125 --body "Issue #{Issue番号} の実装が完了し、マージされました。"`

7.  **後処理**
    *   ローカルの `main` ブランチに切り替え、最新の状態に更新してください。
    *   `git checkout main && git pull`

8.  **完了報告**
    *   全ての処理が完了したことをユーザーに伝えてください。
